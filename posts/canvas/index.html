HTML Canvas의 역사

HTML Canvas는 웹에서 그래픽스, 애니메이션, 게임 등을 그리기 위한 HTML 요소입니다. 이 요소는 2D 그리기 콘텍스트나 WebGL을 이용한 3D 렌더링을 지원합니다.

Canvas 기술은 약 2004년부터 Apple이 macOS의 Dashboard 위젯용으로 개발하기 시작했습니다. 이후에는 Firefox, Opera 등의 브라우저에도 도입되었고, 2008년에 HTML5에 포함되어 널리 사용되게 되었습니다.

HTML Canvas

HTML Canvas 요소는 웹 페이지에 2D, WebGL과 같은 3D 렌더링 컨텍스트를 지원합니다.

Canvas의 특징:

  • 웹 페이지에서 2D 그래픽을 그리위 위한 HTML 요소.
  • JavaScript를 사용하여 동적으로 그림 그리기.
  • WebGL을 사용하여 Canvas 요소 위에서 3D 그래픽스 렌더링 가능.

간단한 사용법

HTML Canvas를 사용하기 위해서는 먼저 HTML 문서에 <canvas> 요소를 추가합니다.

<canvas id="myCanvas" width="200" height="200"></canvas>

그 다음, JavaScript를 사용하여 캔버스에 그림을 그릴 수 있습니다.

// 캔버스 요소를 선택하고 2D 콘텍스트를 얻음
var canvas = document.getElementById('myCanvas');
var ctx = canvas.getContext('2d');

// 사각형 그리기 (x, y, 너비, 높이)
ctx.fillStyle = 'red';
ctx.fillRect(50, 50, 100, 100);

// 직선 그리기
// 직선의 시작점 설정
ctx.moveTo(50, 50);

// 직선의 끝점 설정
ctx.lineTo(350, 350);

// 직선 그리기
ctx.stroke();

// 원 그리기
ctx.beginPath();
ctx.arc(100, 100, 50, 0, 2 * Math.PI);
ctx.stroke();
// 원의 중심의 X 축, Y축, 반지름 길이, 시작 각도, 종료 각도. 라디안 사용

Canvas Animation

캔버스(Canvas)를 사용하여 웹에서 애니메이션을 만들 수 있습니다. 이는 주로 JavaScript와 캔버스의 2D 렌더링 컨텍스트를 활용하여 이루어집니다. 캔버스 애니메이션을 만드는 기본적인 과정은 다음과 같습니다:

  1. 초기 설정: HTML에 <canvas> 요소를 추가하고, JavaScript에서 해당 요소의 2D 렌더링 컨텍스트를 얻습니다.
// HTML
<canvas id="myCanvas" width="400" height="400"></canvas>

// javascript
var canvas = document.getElementById('myCanvas');
var ctx = canvas.getContext('2d');
  1. 애니메이션 루프 생성: 애니메이션을 만들려면 반복적으로 캔버스를 다시 그려야 합니다. 이를 위해 requestAnimationFrame 함수를 사용하여 애니메이션 루프를 생성합니다.
// javascript
function animate() {
    // 애니메이션 코드...
    
    requestAnimationFrame(animate);
}
animate();
  1. 캔버스 지우기: 각 프레임 시작 시점에 전체 캔버스를 지워 이전 프레임의 그림이 남아있지 않게 합니다.
// javascript
ctx.clearRect(0, 0, canvas.width, canvas.height);
  1. 업데이트 & 그리기: 애니메이션의 상태를 업데이트하고 캔버스에 다시 그립니다. 예를 들어 움직이는 원을 그리는 애니메이션을 만든다고 가정하면:
var x = canvas.width / 2;
var y = canvas.height / 2;
var radius = 50;
var speedX = 2;
var speedY = 2;

function animate() {
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    
    // 원의 위치 업데이트
    x += speedX;
    y += speedY;

    // 원 그리기
    ctx.beginPath();
    ctx.arc(x, y, radius, 0, 2 * Math.PI);
    ctx.fill();

    // 끝에 닿았을 때 방향 바꾸기
    if (x + radius > canvas.width || x - radius < 0) {
        speedX = -speedX;
    }

    if (y + radius > canvas.height || y - radius < 0) {
        speedY = -speedY;
    }
    
    requestAnimationFrame(animate);
}
animate();

이 방법을 사용하여 캔버스에서 다양한 애니메이션 효과를 구현할 수 있습니다.

Canvas text

HTML5 캔버스에서 텍스트를 그리는 기능은 fillText와 strokeText 메서드를 사용하여 구현됩니다. 이 두 메서드는 텍스트를 캔버스에 직접 그릴 수 있게 해줍니다.

1. 기본 사용법

const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

ctx.font = '30px Arial'; // 텍스트의 글꼴과 크기 설정
ctx.fillText('Hello, Canvas!', 50, 50); // (50, 50) 위치에 'Hello, Canvas!' 텍스트 그리기

2. 텍스트 스타일 설정

텍스트의 스타일을 다양하게 설정할 수 있습니다:

ctx.fillStyle = 'red'; // 텍스트의 색상 설정
ctx.font = 'bold 30px Arial'; // 텍스트의 글꼴, 스타일 및 크기 설정
ctx.fillText('Hello, Canvas!', 50, 50);

3. 텍스트 테두리 그리기

텍스트의 테두리만 그리려면 strokeText 메서드를 사용합니다:

ctx.strokeStyle = 'blue'; // 텍스트 테두리의 색상 설정
ctx.lineWidth = 2; // 테두리 선의 너비 설정
ctx.strokeText('Hello, Canvas!', 50, 50);

4. 텍스트 측정

텍스트의 너비를 측정할 때는 measureText 메서드를 사용합니다. 이 메서드는 텍스트의 너비를 반환하는 객체를 제공합니다:

const metrics = ctx.measureText('Hello, Canvas!');
console.log(metrics.width); // 'Hello, Canvas!'의 너비 출력

이를 통해 텍스트의 정확한 너비를 알 수 있어, 레이아웃이나 텍스트를 중앙에 배치하는 등의 작업에 유용합니다.

5. 텍스트 정렬과 기준선

텍스트의 수평 정렬과 수직 기준선을 설정할 수 있습니다:

ctx.textAlign = 'center'; // 'start', 'end', 'left', 'right', 'center' 중 하나
ctx.textBaseline = 'middle'; // 'top', 'hanging', 'middle', 'alphabetic', 'ideographic', 'bottom' 중 하나

ctx.fillText('Centered Text', canvas.width / 2, canvas.height / 2);

위의 코드는 캔버스의 중앙에 텍스트를 정확하게 위치시킵니다.

이렇게 캔버스에서 텍스트를 그릴수 있습니다.

캔버스 사용시 주의 사항

  1. 접근성 (Accessibility): 캔버스 요소는 기본적으로 스크린 리더와 같은 보조 기술에 내용을 전달하지 않습니다. 이는 시각 장애인 사용자가 캔버스의 내용을 이해하기 어렵게 만듭니다. 해결책으로 ARIA 및 offscreen 요소를 사용하여 캔버스의 내용을 설명할 수 있습니다.

  2. 성능: 많은 그래픽 연산이나 복잡한 애니메이션을 진행하면 브라우저의 성능에 영향을 줄 수 있습니다. requestAnimationFrame을 사용하여 애니메이션을 최적화하고 불필요한 렌더링을 피하는 것이 중요합니다.

  3. 해상도 및 스케일링: 캔버스의 물리적 크기와 그릴 때 사용하는 해상도는 다를 수 있습니다. 화면에 따라 적절하게 스케일링하고, 레티나 디스플레이처럼 고해상도 디스플레이에서도 잘 보이게 하려면 주의가 필요합니다.

  4. 메모리 사용: 많은 이미지나 데이터를 캔버스에 로드할 경우 메모리 사용량이 증가할 수 있습니다. 불필요한 객체는 메모리에서 해제해 주는 것이 좋습니다.

  5. 보안: getImageData와 같은 메서드를 사용하여 픽셀 데이터에 접근할 때 보안에 관한 문제가 발생할 수 있습니다. 예를 들어, 다른 도메인에서의 이미지를 캔버스에 로드한 후 픽셀 데이터에 접근하려고 하면 보안 정책에 위반될 수 있습니다.

  6. 이미지 품질: toDataURL을 사용하여 캔버스의 내용을 이미지로 변환할 때, 기본적으로 JPEG 혹은 PNG 형식으로 변환됩니다. 필요에 따라 적절한 이미지 품질과 포맷을 선택해야 합니다.

  7. 코드 복잡성: 복잡한 그래픽이나 애니메이션을 만들 때 코드가 복잡해질 수 있습니다. 코드의 모듈성과 재사용성을 유지하기 위해 적절한 구조와 디자인 패턴을 적용하는 것이 좋습니다.
