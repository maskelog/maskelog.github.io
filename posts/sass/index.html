Sass란 무었인가?

Sass(Syntactically Awesome StyleSheets)는 CSS로 컴파일 되는 스타일 시트 확장 언어이고 CSS 전처리기의 하나입니다.
브라우저가 Sass를 직접 로딩하는것이 아닌 Sass로 개발을 하고 CSS로 익스포트 하는 과정을 거칩니다.
브라우저가 직접 Sass를 읽는것이 아닌 CSS를 컴파일 하여 CSS 파일을 생성합니다.

Sass를 사용하는 이유?

  1. 변수 사용: CSS에서는 기본적으로 변수를 지원하지 않습니다. Sass에서는 변수를 정의하여 스타일시트 전체에 걸쳐 색상, 폰트 크기, 여백 등의 값을 일관되게 유지할 수 있습니다.

  2. 중첩(Nesting): Sass에서는 선택자의 중첩을 허용하여, CSS의 구조를 더 명확하고 읽기 쉽게 만들 수 있습니다.

  3. 믹스인(Mixins): 믹스인을 사용하면 CSS 선언을 재사용 가능한 방식으로 정의하고 여러 곳에 적용할 수 있습니다. 이를 통해 반복 코드를 줄이고 코드의 재사용성을 높일 수 있습니다.

  4. Extend/Inheritance: 기존의 클래스 스타일을 다른 클래스에 상속하여 코드 중복을 줄일 수 있습니다.

  5. 조건문과 반복문: Sass는 if, for, each와 같은 제어 지시문을 제공하여, 복잡한 스타일시트를 더 유연하게 만들 수 있습니다.

  6. 함수: 내장 함수(색상 조절, 문자열 조작 등)를 제공하여 작업 효율성을 향상시킵니다. 사용자 정의 함수 생성도 가능합니다.

  7. 파일 분할 및 가져오기: Sass의 @import 지시문을 사용하여 여러 Sass 파일을 하나의 CSS 파일로 컴파일 할 수 있습니다. 이를 통해 프로젝트 구조를 더 모듈화하고 관리하기 쉽게 만들 수 있습니다.

  8. 커뮤니티 및 확장성: Sass는 넓은 사용자 기반이 있으며, 많은 라이브러리와 플러그인이 있어 확장성이 뛰어납니다.

  9. 성능 최적화: 불필요한 선택자나 속성을 제거하고, 효율적인 CSS를 생성하여 웹사이트의 로딩 성능을 향상시킬 수 있습니다.

  10. 컴파일 시 오류 확인: Sass를 컴파일 할 때 문제점이나 오류가 있으면 즉시 알려주므로, 오류를 빠르게 수정할 수 있습니다.

Sass는 이러한 기능들을 통해 CSS 작업의 복잡성을 줄이고, 큰 프로젝트에서도 스타일시트를 쉽게 관리할 수 있게 도와줍니다. 따라서 많은 웹 개발자들이 Sass와 같은 CSS 전처리기를 선호합니다.

Sass 사용하는법

https://sass-lang.com/install/

윈도우 환경에서는 Chocolatey 패키지 관리자를 통해 Dart Sass 설치와 VScode 익스텐션을 사용하는 방법이 있습니다.
저는 VScode에서 Live Sass Compiler를 사용하였습니다.

Live Sass Compiler 사용법은 VScode하단 Status Bar에 Watch Sass라는 버튼이 생성되는데 여기를 누르고 Sass버튼을 누르고 sass를 저장하면 자동으로 컴파일이 되어 CSS파일을 생성해줍니다.

Sass의 주요 특징

  • Sass에서 변수 사용 Sass에서는 ‘$’ 기호를 사용하여 변수를 정의합니다.
$primary-color: #333;
body {
  background-color: $primary-color;
}
  • 중첩: 선택자의 중첩을 지원합니다.
nav {
  ul {
    margin: 0;
    padding: 0;
    list-style: none;
  }
  li { display: inline-block; }
  a {
    display: block;
    padding: 6px 12px;
    text-decoration: none;
  }
}
  • 믹스인: 코드를 재사용하게 해주는 믹스인을 정의하고 @include를 사용하여 적용할 수 있습니다.
@mixin transform($property) {
  -webkit-transform: $property;
  -ms-transform: $property;
  transform: $property;
}
.box { @include transform(rotate(30deg)); }
  • 확장/상속: 다른 선택자의 스타일을 상속받을 수 있습니다.
.message {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}
.success {
  @extend .message;
  border-color: green;
}
  • 연산: 산술 연산자를 사용하여 값을 계산할 수 있습니다.
.container {
  width: 100%;
  padding: 10px - 2px * 3;
}

  • 함수: 색상 조절, 문자열 조작 등의 내장 함수가 있으며 사용자 정의 함수도 작성할 수 있습니다.

  • 조건문 & 반복문: @if, @for, @each, @while 등의 제어 지시문을 사용하여 더 복잡한 로직을 작성할 수 있습니다.

  • 모듈성: 여러 Sass 파일로 코드를 분할하고, @import 또는 최신의 @use 및 @forward 지시어를 사용하여 하나의 스타일시트로 병합할 수 있습니다.

Sass의 장점

Sass는 CSS 작업을 더욱 효율적이고 구조화된 방식으로 처리할 수 있게 도와줍니다. 복잡한 웹 프로젝트나 큰 규모의 애플리케이션에서 CSS 코드의 유지 보수와 확장성을 향상시키는데 큰 도움이 됩니다.

Sass는 CSS를 더욱 강력하게 만들며, 개발자들이 더 나은 코드 구조를 가진, 재사용 가능하고, 유지 관리가 쉬운 스타일시트를 작성할 수 있게 합니다.
